- 정의
	- 프로젝트 설계(Project Design)란 문제를 분석하고, 이를 해결하기 위한 전체 구조와 흐름을 계획하는 작업.
- 설계 목적
	- 문제를 명확히 이해기 위함
	- 개발 중 혼란을 줄이고, 재사용성을 높이기 위함
	- 협업이나 유지보수를 가능하게 만들기 위함


# 프로젝트 설계 5단계 프레임워크
## 1. 문제 정의 (Problem Definition)
- 무엇이 문제? 어떤 부분이 불편?
- 사용자는 누구? 사용하는 집단은 어디?
- 어떤 행동을 어떻게 하고 싶은가? 어떤 문제를 어떻게 해결할 것인가?

## 2. 요구사항 정리 (Requirement Analysis)
- 앞서 정의한 문제에서 구체적으로 어떤 기능이 필요한지?
- 이 단계에서 MVP(Minimum Viable Product)를 생각할 것
	- **최소한 이건 돼야 한다**

## 3. 아키텍처 설계 (System Design)
- 큰 그림(컴포넌트 간 관계, 흐름, 저장소)
	- 주요 구성 요소 정의 (LLM, DB, API, Frontend)
	- 데이터 흐름 (Input -> 처리 -> output)
	- 통신 방식 (Sync/Async, REST, WebSocket)
- 도구 추천: Lucidchart, Whimsical, Miro

## 4. 세부 설계 (Detail Design)
- 각 기능마다 어떻게 동작할지 구체화
	- 어떤 라이브러리?
	- 어떤 모델?
	- 어떤 프롬프트?
	- 어떤 파라미터?
	- 어떤 데이터를 어떻게 저장?조회?

## 5. 리스크 검토 및 테스트 계획
- 어떤 부분이 실패할 수 있는지?
- 어떻게 테스트할 것인지?
- 입력값, 검증, 모델 오답, 성능 이슈 등 예측
