# 작업 시작 ~ 커밋
|목적|명령어|설명|
|---|---|---|
|저장소 초기화|`git init`|현재 폴더를 Git 저장소로 만든다.|
|상태 확인|`git status`|어떤 파일이 변경/추가/삭제되었는지 확인.|
|파일 스테이징|`git add <파일명>`|커밋할 파일 선택. (`git add .`은 전체 추가)|
|커밋 생성|`git commit -m "메시지"`|현재 스테이징된 변경을 하나의 커밋으로 기록.|
|로그 보기|`git log`|커밋 히스토리 확인.|
|변경사항 비교|`git diff`|수정된 내용 확인. (`git diff --staged`는 스테이징된 변경 비교)|


# branch
|목적|명령어|설명|
|---|---|---|
|브랜치 목록 보기|`git branch`|현재 로컬 브랜치 목록 확인.|
|새 브랜치 생성|`git branch <브랜치명>`|새 브랜치를 만든다.|
|브랜치 이동|`git checkout <브랜치명>`|다른 브랜치로 전환.|
|생성 + 이동 동시에|`git checkout -b <브랜치명>`|브랜치 만들고 바로 이동.|
|브랜치 삭제|`git branch -d <브랜치명>`|브랜치 삭제 (`-D`는 강제삭제).|


# 병합 / 원격 동기화
| 목적                   | 명령어                           | 설명                       |
| -------------------- | ----------------------------- | ------------------------ |
| 원격 저장소 등록            | `git remote add origin <URL>` | GitHub 등 원격 repo와 연결.    |
| 원격 저장소 확인            | `git remote -v`               | 연결된 원격 repo 주소 보기.       |
| 최신 변경 가져오기           | `git fetch`                   | 원격 커밋 정보만 가져온다 (병합 안 함). |
| pull (fetch + merge) | `git pull`                    | 원격 변경을 현재 브랜치로 병합.       |
| push                 | `git push origin <브랜치명>`      | 로컬 커밋을 원격으로 업로드.         |
| 브랜치 병합               | `git merge <브랜치명>`            | 다른 브랜치의 커밋을 현재 브랜치에 병합.  |


# 되돌리기 / 수정 / 복구
|목적|명령어|설명|
|---|---|---|
|변경 취소 (작업 디렉토리)|`git restore <파일>`|수정된 파일을 최근 커밋 상태로 되돌림.|
|전체 취소|`git restore .`|모든 변경 취소.|
|스테이징 해제|`git restore --staged <파일>`|`git add` 한 걸 취소.|
|커밋 되돌리기|`git reset --soft HEAD~1`|마지막 커밋만 되돌리고 변경 유지.|
|강제 되돌리기|`git reset --hard HEAD~1`|커밋과 변경 모두 삭제. ⚠️|
|특정 파일 checkout|`git checkout -- <파일>`|예전 방식 (`git restore`와 동일).|


# stash
|목적|명령어|설명|
|---|---|---|
|임시 저장|`git stash`|현재 변경사항을 임시로 보관.|
|목록 보기|`git stash list`|보관된 stash 목록 확인.|
|복원|`git stash pop`|가장 최근 stash 복원 + 삭제.|
|여러 개 있을 때 선택 복원|`git stash apply stash@{2}`|지정 stash 복원.|
|삭제|`git stash drop stash@{0}`|특정 stash 삭제.|
|전체 삭제|`git stash clear`|모든 stash 삭제.|


# etc
| 목적           | 명령어                                  | 설명                     |
| ------------ | ------------------------------------ | ---------------------- |
| 마지막 커밋 수정    | `git commit --amend`                 | 메시지나 파일 수정 후 커밋 다시 작성. |
| 원격 브랜치 목록    | `git branch -r`                      | 원격에 있는 브랜치 확인.         |
| 로컬+원격 전체 브랜치 | `git branch -a`                      | 로컬/원격 브랜치 모두 보기.       |
| 특정 커밋으로 되돌리기 | `git checkout <커밋ID>`                | 특정 시점의 파일로 돌아가기.       |
| 원격과 차이 보기    | `git fetch` → `git diff origin/main` | 로컬 vs 원격 비교.           |
