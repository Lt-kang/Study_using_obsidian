# 개념
- 항목별 크기(빈도/비용/불량 수/시간 낭비 등)를 내림차순 막대 그래프로 표시하고, 
  그 막대를 왼쪽부터 누적해가며 누적비율을 꺽은선으로 함께 표시한 그래프
## 사용 목적
- 중요한 소수(critical few)가 전체 영향의 대부분을 만든다.
- `어디부터 개선해야 가장 효과적인가?`를 찾는 것이 핵심 목적
	- pareto principle(law of the vital few, principle of factor sparsity)
		- 소수의 중요한 원인(20%)이 결과의 대부분(80%)을 만들어낸다는 법칙

# 특징
- **정렬**이 핵심
	- 반드시 항목이 큰 것부터 작은 것으로 정렬해서 보여줌
- 막대 + 누적선의 결합
	- 막대: 각 원인의 "크기"
	- 누적선: "상위 몇 개까지 합치면 전체의 몇 %인가"
	- **어디까지가 핵심 원인인가**
- **집중 대상 선정 도구**
	- pareto chart는 인과관계 증명 불가능
	- 어떤 항목부터 손대면 효과가 큰지 정할 수 있음
- 단위: 건수 / 비용 / 시간 / 손실
	- ex) 불량 건수 -> 불량으로 인한 비용
	  위와 같이 기준을 변경할 경우
	  진짜로 중요한 문제가 달라질 수 있음.

# 장점
- 우선 순위가 직관적
- 의사결정/보고에 강함
- 리소스 최적화에 도움
- 개선 전/후 비교가 쉬움

# 단점
- 인과관계를 보여주지 못함
- 분류 기준에 따라 결론이 바뀜
- 데이터가 불완전하면 위험
- 시간 흐름을 직접 보여주지 못함
- 상위 원인이 많거나 값이 비슷하면 메세지가 약함

# 구성
- 막대그래프 + 누적선 그래프
## X축
- 문제의 원인/항목
- 중요도가 큰 순서대로 내림차순 정렬

## Y축
- 왼쪽: 발생 빈도 / 비용 / 불량 수 등의 절대값
- 오른쪽: 누적 비율(%)

## 그래프
- 막대 그래프: 각 항목의 크기(빈도 or 비용 등)
- 꺽은선 그래프: 누적 비율


# 사용 적합한 상황
- 불량/장애/사고 원인 중 무엇부터 해결할 지
- 고객 불만/CS 이슈 중 어디가 더 큰 비중인지
- 비용 절감 포인트 찾기
- 업무 비효율/시간 낭비 요인 파악
- 매출/성과 기여도 파악

# 사용 부적합한 상황
- 시간에 따른 변화(추세)가 핵심일 때
	- 선그래프/관리도/추세분석이 더 적합
- 인과관계를 증명해야 할 때
	- 회귀/실험/원인분석(5 Why, 어골도 등)이 필요
- 항목이 너무 많고 분류가 불명확할 때
	- 먼저 분류 체계를 정리해야 함

# 활용 분야
- 품질 관리 (불량 원인 분석)
- 경영/마케팅 (매출 기여도 분석)
- 고객 불만 분석
- 업무 우선순위 설정
- 시간 관리 (비효율 요소 파악)
